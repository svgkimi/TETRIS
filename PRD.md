## 1. 프로젝트 개요

### 1.1 목적
브라우저에서 별도 설치 없이 즉시 즐길 수 있는 **최신 가이드라인(Tetris Guideline) 기반의 테트리스 게임**을 만든다. 단순히 블록을 쌓는 클래식 경험을 넘어, 부드러운 조작감(Responsive Input), 화려한 이펙트(Juice), 그리고 현대 테트리스의 표준 규칙(SRS, Hold, Ghost Piece 등)을 충실히 구현하여 캐주얼 유저와 코어 퍼즐 게임 유저 모두를 만족시키는 것을 목표로 한다.

### 1.2 타겟 유저
| 구분 | 설명 |
|---|---|
| **캐주얼 유저** | 짧은 시간 동안 가볍게 즐기고 싶은 일반 웹 유저. 별도 튜토리얼 없이도 직관적으로 플레이 가능해야 함 |
| **코어 퍼즐 게이머** | 테트리스 99, 테트리오(Tetr.io), 테트리스 이펙트 등을 경험한 유저. SRS, 홀드, 콤보 등 표준 규칙이 정확히 구현되어 있길 기대함 |
| **레트로 감성 유저** | 과거 오리지널 테트리스에 향수가 있는 유저. 친숙한 조작감 + 업그레이드된 비주얼에 만족을 느낌 |

### 1.3 핵심 가치 제안 (Value Proposition)
- **즉시성**: 로딩 없이 브라우저 접속 즉시 플레이
- **정통성**: 공식 가이드라인 규칙 준수로 코어 유저의 신뢰 확보
- **쾌감(Juice)**: 라인 클리어, 콤보, 테트리스(4줄 클리어) 시 화려한 이펙트와 사운드로 몰입감 극대화

## 2. 최신 테트리스 핵심 룰 (SRS 기반)

### 2.1 SRS (Super Rotation System)
- 테트리미노 회전 시 공식 SRS 규정에 따른 **회전 상태(0, R, 2, L)** 를 유지한다.
- 회전이 충돌로 실패할 경우, 규정된 **킥 테이블(Kick Table)** 을 순차 적용하여 벽/바닥/다른 블록과의 근접 상황에서도 자연스러운 회전(Wall Kick, Floor Kick)을 지원한다.
  - 일반 블록(J, L, S, T, Z): 표준 5종 킥 데이터 테이블 적용
  - I 블록: 별도의 I 블록 전용 킥 테이블 적용
  - O 블록: 회전 시 위치 변화 없음(킥 불필요)
- 회전 시 **T-Spin 판정**의 기초가 되는 코너 점유 규칙(3-corner rule)을 판정 로직에 포함한다.

### 2.2 홀드 (Hold) 기능
- 플레이 중 현재 테트리미노를 홀드 슬롯에 보관하고, 보관 중이던 블록(또는 다음 블록)을 즉시 꺼내 사용할 수 있다.
- 하나의 테트리미노가 필드에 등장한 후, **홀드는 1회만 사용 가능**하며 해당 블록이 필드에 고정(Lock)되기 전까지는 다시 홀드할 수 없다(연속 홀드 스왑 방지).
- 홀드 슬롯 UI는 필드 좌측(또는 우측, 레이아웃에 따라 결정)에 항상 노출되어 현재 보관된 블록 모양을 표시한다.

### 2.3 고스트 피스 (Ghost Piece)
- 현재 조작 중인 테트리미노가 **그대로 하드 드롭될 경우 도달할 최종 위치**를 필드 하단에 반투명/윤곽선 형태로 실시간 미리 표시한다.
- 좌우 이동, 회전 시 고스트 피스 위치는 매 프레임 즉시 재계산되어 갱신된다.
- 시각적으로 실제 블록과 명확히 구분되는 스타일(투명도, 테두리 강조 등)을 적용한다.

### 2.4 하드 드롭 (Hard Drop)
- **Spacebar** 입력 시, 현재 테트리미노는 고스트 피스 위치까지 즉시 낙하하며 그 자리에 즉시 고정(Lock)된다.
- 하드 드롭은 낙하 거리에 비례한 **점수 보너스**를 지급한다(예: 셀당 2점).
- 하드 드롭 시 낙하 궤적을 강조하는 잔상/충격 이펙트를 재생하여 타격감을 부여한다.
- 이 외에 **소프트 드롭(하단 방향키, 셀당 1점)** 도 기본 조작으로 지원한다.

### 2.5 기타 표준 규칙
- **넥스트 큐(Next Queue)**: 다음에 등장할 테트리미노를 최소 3~5개 미리보기로 제공한다.
- **7-bag 랜덤 시스템**: 7종 테트리미노가 한 세트(bag)로 묶여 무작위 순서로 등장하며, 같은 블록이 지나치게 연속 출현하지 않도록 보장한다.
- **락 다운(Lock Down)**: 블록이 바닥/장애물에 닿은 후 일정 유예시간(Lock Delay, 기본 0.5초) 동안 이동/회전이 가능하며, 유예시간 종료 시 고정된다.

## 3. 핵심 기능 목록

### 3.1 게임 흐름 제어
- **시작(Start)**: 타이틀/메인 화면에서 게임 시작 트리거
- **일시정지(Pause)**: 플레이 중 언제든 일시정지 가능 (ESC 또는 지정 키), 일시정지 중 필드는 블러 처리되어 노출되지 않음
- **재시작(Restart)**: 게임 오버 화면 또는 일시정지 메뉴에서 즉시 재시작 가능

### 3.2 점수 및 레벨업 시스템
| 항목 | 규칙 |
|---|---|
| 라인 클리어 점수 | 1줄: 100점, 2줄: 300점, 3줄: 500점, 4줄(테트리스): 800점 (레벨 배수 적용) |
| T-Spin 보너스 | 일반 클리어 대비 가중치 부여 (예: T-Spin Double 1200점 등) |
| 레벨 시스템 | 일정 라인 수(예: 10라인) 클리어 시 레벨업, 레벨이 오를수록 블록 낙하 속도 증가 |
| 낙하 점수 | 소프트 드롭 셀당 1점, 하드 드롭 셀당 2점 |

### 3.3 콤보 시스템
- 라인 클리어가 **연속으로 발생**할 때마다 콤보 카운트가 1씩 증가하며, 콤보 수에 비례한 추가 점수를 지급한다.
- 콤보가 끊기지 않고 이어질 경우(Back-to-Back), 테트리스나 T-Spin이 연속될 때 추가 보너스 배율을 적용한다(Back-to-Back 시스템).
- 콤보 진행 상황은 화면에 실시간 카운터와 이펙트(콤보 수 텍스트 팝업 등)로 시각화한다.

### 3.4 부가 기능
- 최고 점수(하이스코어) 로컬 저장 및 표시
- 사운드 On/Off 토글 및 배경음악/효과음 지원
- 키 조작 안내(조작법 가이드) UI

## 4. 사용자 시나리오

### 4.1 메인 플로우: 접속 → 플레이 → 게임오버 → 재시작

1. **접속 및 진입**
   - 유저가 웹사이트에 접속하면 타이틀 화면(로고, "시작하기" 버튼, 조작법 안내)이 노출된다.
2. **게임 시작**
   - "시작하기" 클릭 또는 지정 키 입력 시 필드가 초기화되고 카운트다운(3, 2, 1) 후 첫 테트리미노가 등장한다.
3. **플레이 중**
   - 유저는 좌우 이동, 회전(SRS), 소프트/하드 드롭, 홀드를 사용해 블록을 배치한다.
   - 라인이 완성되면 클리어 이펙트(플래시, 파티클)와 함께 점수/콤보가 갱신된다.
   - 일정 라인 클리어 시 레벨업 연출과 함께 낙하 속도가 증가한다.
   - 유저는 언제든 일시정지하여 재개하거나 종료할 수 있다.
4. **게임 오버**
   - 새로 등장한 테트리미노가 필드 상단에서 즉시 다른 블록과 겹쳐 배치 불가능한 상태(Block Out / Top Out)가 되면 게임 오버가 발생한다.
   - 게임 오버 연출(필드 어두워짐, "GAME OVER" 텍스트, 최종 점수 및 최고 기록 갱신 여부 표시)이 재생된다.
5. **재시작 또는 종료**
   - 게임 오버 화면에서 "다시하기" 버튼 클릭 시 3번(게임 시작) 단계로 즉시 복귀한다.
   - "메인으로" 버튼 클릭 시 타이틀 화면으로 복귀한다.

### 4.2 예외 시나리오
- **일시정지 중 이탈**: 브라우저 탭 비활성화(포커스 아웃) 시 자동으로 일시정지 처리되어 불이익 없이 재개 가능하다.
- **조작 불가 상태**: 게임 오버/일시정지 상태에서는 게임 입력(이동/회전/드롭)이 무시되고 메뉴 관련 입력만 활성화된다.

## 5. 기술 스택 및 아키텍처 요구사항

### 5.1 기술 스택
- **프론트엔드**: React + TypeScript (Vite 빌드 환경)
- **렌더링**: HTML5 Canvas API (또는 최적화된 CSS Grid) — 60FPS 유지를 최우선으로 판단해 선택
- **스타일링**: Tailwind CSS (또는 Vanilla CSS)
- **상태 관리**: React 로컬 상태 + 필요 시 경량 상태 관리 라이브러리(Zustand 등) 검토, 게임 루프 자체는 React 상태와 분리된 별도 엔진에서 처리
- **오디오**: Web Audio API 기반 효과음/BGM 재생

### 5.2 아키텍처 요구사항
- **관심사 분리**: 게임 코어 엔진(회전, 충돌 판정, 라인 삭제, 점수 계산 등 순수 로직)과 React UI 렌더링 레이어를 명확히 분리한다. 코어 엔진은 프레임워크에 종속되지 않는 순수 TypeScript 모듈로 구현한다.
- **게임 루프**: `requestAnimationFrame` 기반의 고정 타임스텝(Fixed Timestep) 루프를 사용해 프레임 드랍 환경에서도 게임 로직의 일관성을 보장한다.
- **타입 안정성**: 테트리미노 형태/좌표/회전 상태, 게임 상태(Ready, Playing, Paused, GameOver)를 명확한 TypeScript interface/type으로 정의한다.
- **성능 최적화**: 불필요한 React 리렌더링을 최소화하기 위해 렌더링이 빈번한 필드/블록 정보는 Canvas 직접 제어 또는 메모이제이션(memo, useMemo/useCallback)을 적극 활용한다.
- **반응형 디자인**: 데스크톱 브라우저를 우선 지원하되, 태블릿 이상 화면 크기에서도 레이아웃이 깨지지 않도록 대응한다.

## 6. 단계별 개발 우선순위 (Milestones)

### Milestone 1 — 코어 게임 엔진 (Core Engine)
- 게임 필드(그리드) 데이터 구조 설계
- 7종 테트리미노 정의 및 7-bag 랜덤 큐 구현
- 기본 이동/회전(SRS 킥 테이블 포함)/충돌 판정 로직
- 라인 클리어 판정 및 삭제 로직
- 게임 루프(자동 낙하, Lock Delay) 구현

### Milestone 2 — 기본 플레이 가능 버전 (Playable MVP)
- Canvas/Grid 기반 필드 렌더링
- 키보드 입력 처리(이동, 회전, 소프트/하드 드롭)
- 고스트 피스 표시
- 홀드 기능 및 넥스트 큐(3~5개) UI
- 시작/일시정지/재시작/게임 오버 플로우 연결

### Milestone 3 — 점수/레벨/콤보 시스템
- 점수 계산 로직(라인 클리어, 드롭 보너스, T-Spin 판정)
- 레벨업 및 낙하 속도 증가 로직
- 콤보 및 Back-to-Back 시스템
- 하이스코어 로컬 저장(LocalStorage)

### Milestone 4 — 비주얼 & 사운드 폴리싱 (Juice)
- 라인 클리어/테트리스/콤보 파티클 및 화면 흔들림(Screen Shake) 이펙트
- 하드 드롭 잔상 효과
- 효과음 및 배경음악 추가
- 레벨업/게임오버 연출 강화

### Milestone 5 — 마무리 및 최적화
- 60FPS 성능 프로파일링 및 최적화
- 반응형 레이아웃 대응
- 조작법 가이드 UI, 접근성(키 리매핑 등) 검토
- 버그 픽스 및 QA
